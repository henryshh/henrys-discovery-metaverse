# Antigravity Semantic Knowledge Base (语义知识库)

本知识库整合了《运用 AI 与数字孪生革新汽车拧紧质量控制》报告中的核心理念，作为 Antigravity 项目后续开发的指导标准。

## 1. 核心方法论 (Core Methodologies)
*   **仿生推理链 (CoE Chain / Bionic Inference)**: 将高级工程师的“观察-假设-验证-决策”逻辑数字化。系统不仅给出结果，更要重现诊断思维过程。
*   **可解释性 AI (XAI - Explainable AI)**: 拒绝“黑盒”。AI 必须能定位并解释曲线异常的物理原因（如：第 3 圈丝牙损坏）。
*   **合成数据 (Synthetic Data)**: 利用数字孪生模拟失效场景生成样本，解决工业现场“负样本”稀缺的问题，实现 AI 的预训练。

## 2. 系统架构 (System Architecture)
*   **边云协同 (Edge-Cloud Synergy)**: 
    *   **Edge (边缘侧)**: 毫秒级实时监控与干预。
    *   **Cloud (云端)**: 跨工位、跨工厂的全局模型迭代与大数据分析。
*   **多模态融合 (Multi-Modal Fusion)**: 整合 **[拧紧曲线] + [视觉位姿] + [电机电流] + [声振信号]**，实现全方位的质量感知。

## 3. 数字孪生定义 (Digital Twin Definitions)
*   **物理实体镜像 (Mirroring)**: 实现生产现场的实时可视化。
*   **孪生档案 (DT Archive)**: 为每个 **VIN (车架号)** 的每颗螺栓建立“数字出生证明”，存储 100% 的原始特征。
*   **预测性维护 (PdM)**: 通过 AI 监测设备性能衰减趋势，在故障发生前触发维护申请。
